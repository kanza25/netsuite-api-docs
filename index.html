NetSuite Integration API Documentation

OAuth 1.0 Authentication & Integration Endpoints

Overview

This documentation provides comprehensive details for integrating with NetSuite's endpoints. These APIs enable you to retrieve invoice and payment data from your NetSuite environment.

All endpoints require OAuth 1.0 authentication and are designed to work seamlessly with API clients like Postman. The APIs return data in JSON format and support standard HTTP response codes.

Available Endpoints:

  • Invoice Payments: Retrieve invoices with their associated payment records
  • Invoices: Retrieve invoice records with detailed line item information

Before You Begin:

  • Ensure you have valid OAuth 1.0 credentials (Consumer Key, Consumer Secret, Token ID, Token Secret)
  • Configure authentication in your API client (Postman recommended)
  • Manually add required headers (Content-Type and Accept) to each request

OAuth 1.0 Configuration for Postman

All NetSuite integration endpoints require OAuth 1.0 authentication. Configure the following parameters in Postman before making any API requests:

⚠️ Important:

Keep these credentials secure. Never commit them to version control or share them publicly.

Parameter Value Description
Auth Type OAuth 1.0 Authentication method
Account 11375429_SB2 NetSuite account ID (Sandbox)
Consumer Key String OAuth consumer key from NetSuite
Consumer Secret String OAuth consumer secret
Access Token (Token ID) String OAuth access token
Token Secret String OAuth token secret
Signature Method HMAC-SHA256 Signature hashing algorithm

Complete OAuth 1.0 Credentials:

{
  "account": "11375429_SB2",
  "consumerKey": "String",
  "consumerSecret": "String",
  "tokenId": "String",
  "tokenSecret": "String",
  "signatureMethod": "HMAC-SHA256"
}

Postman Configuration Steps:

  1. Open your request in Postman
  2. Go to the Authorization tab
  3. Select OAuth 1.0 from the Type dropdown
  4. Fill in the credentials as shown above
  5. Set Add auth data to: "Request Headers"
  6. Enable these checkboxes:
    • ✅ Add empty parameters to signature
    • ✅ Encode the parameters in the Authorization header
  7. IMPORTANT: Go to the Headers tab and manually add:
    • Content-Type: application/json
    • Accept: application/json

Important Notes

  • This is a Sandbox (SB2) environment configuration
  • OAuth 1.0 signatures are automatically generated by Postman
  • The signature method must be set to HMAC-SHA256
  • The Authorization header is automatically added when OAuth 1.0 is configured
  • Headers must be manually added to each request (Content-Type and Accept)
  • All requests require proper OAuth 1.0 authentication to access NetSuite data

Common Error: 400 Bad Request

error code: UNEXPECTED_ERROR
error message: An unexpected SuiteScript error has occurred

This error typically occurs when:

  • OAuth 1.0 credentials are missing or incorrect
  • The Content-Type and Accept headers are not added manually
  • The NetSuite account doesn't have access to the script
  • The script deployment is not active or accessible

Solution:

  • Double-check all OAuth 1.0 credentials are entered correctly
  • Manually add headers in the Headers tab: Content-Type and Accept as application/json
  • Check that both OAuth checkboxes are enabled in Postman

Endpoint Overview

This endpoint retrieves invoice records along with their associated payment transactions. Each invoice includes complete header information and an array of related payment records, showing both deposits and applications to invoices.

Query Parameters
Key Value Description
Script 1353 Script internal ID
deploy 1 Script deployment ID
type InvoicePayments Request type identifier for invoice payments endpoint
Headers (Must be manually added in Postman)
Key Value Description
Content-Type application/json Request content type
Accept application/json Expected response content type
Authorization OAuth oauth_consumer_key="...", oauth_token="..." OAuth 1.0 authentication header (auto-generated by Postman)

⚠️ Important:

You must manually add the Content-Type and Accept headers in the Headers tab. The Authorization header is automatically generated when OAuth 1.0 is configured, but Content-Type and Accept are NOT added automatically.

Responses
200 OK

Successfully retrieved invoice and payment data.

  "type": "string",
  "totalRecords": 0,
  "data": [
    {
      "internalId": "string",
      "orderType": "string",
      "tranDate": "string",
      "asOfDate": "string",
      "period": "string",
      "taxPeriod": "string",
      "type": "string",
      "tranId": "string",
      "entity": "string",
      "entityId": "string",
      "account": "string",
      "accountId": "string",
      "memo": "string",
      "amount": 0,
      "payments": [
        {
          "internalId": "string",
          "tranDate": "string",
          "amount": 0,
          "bankAccountNumber": "string"
        }
      ]
    }
  ]
}
204 No Content

No records found matching the criteria.

Notes

  • The API filters transactions created today (datecreated: 'within', 'today')
  • Each invoice includes an array of associated payments
  • Payment amounts show both the deposit (positive amount) and the application to invoice (negative amount)
  • All dates are returned in DD/MM/YYYY format
  • Timestamps are in ISO 8601 format

Endpoint Overview

This endpoint retrieves detailed invoice records including all line items with their associated classification dimensions (class, department, location). Each invoice provides comprehensive billing information suitable for accounting and reporting purposes.

Query Parameters
Key Value Description
Script 1353 Script internal ID
deploy 1 Script deployment ID
type invoice Request type identifier for invoices endpoint
Headers
Key Value Description
Content-Type application/json Request content type
Accept application/json Expected response content type
Authorization OAuth oauth_consumer_key="...", oauth_token="..." OAuth 1.0 authentication header (auto-generated)

Note: The Authorization header is automatically generated when using OAuth 1.0 authentication in Postman.

Responses
200 OK

Successfully retrieved invoice data.

  "success": true,
  "message": "string",

  "type": "string",
  "totalRecords": 0,

      "invoiceNumber": "string",
      "invoiceDate": "string",
      "dueDate": "string",
      "customer": "string",
      "customerId": "string",
      "memo": "string",

        {
          "lineNumber": "string",

          "class": "string",
          "department": "string",
          "location": "string"
        }
      ]
    }

204 No Content

No records found matching the criteria.

Notes

  • Each invoice includes an array of line items with detailed information
  • Invoice dates and due dates are returned in DD/MM/YYYY format
  • Timestamps are in ISO 8601 format
  • Line items include quantity, rate, and calculated amounts
  • Classification dimensions (class, department, location) are included for each line
